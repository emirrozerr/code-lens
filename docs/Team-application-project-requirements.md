# Team Application Project Plan

## Project Objective

The objective of this project is to design and develop a functional digital application in student teams. The project follows a practical, project-based learning approach, covering the entire lifecycle of digital product development: from idea and requirements analysis to implementation, testing, and final presentation.

Students may complete the project using one of two technological tracks:

- **Full-Code Development Track**
- **Low-Code / No-Code / AI-Code Track**

Both tracks pursue the same learning outcomes: designing, implementing, and presenting a working digital product.

---

## Development Tracks

### Full-Code Development Track

Students develop a web application using traditional programming technologies.

| Layer | Options |
|---|---|
| **Backend** | Java, Python, .NET, Node.js |
| **Frontend** | React, Angular, Vue |
| **Database** | PostgreSQL, MySQL, MongoDB |
| **Version Control** | GitHub / GitLab |

Students are responsible for implementing the application logic, database layer, and user interface.

### Low-Code / No-Code / AI-Code Track

Students build an application using visual development platforms and AI-assisted tools.

**Low-Code / No-Code Platforms:** Bubble, Glide, Adalo, Airtable, Microsoft Power Apps

**Automation Tools:** Zapier, Make

**AI-Assisted Development Tools:** GitHub Copilot, Cursor, ChatGPT, Replit

This track emphasizes digital product design, system integration, automation, and AI-assisted development.

---

## Team Organization

Projects are completed in teams of **3–6 students**. Each team should assign the following roles:

| Role | Responsibilities |
|---|---|
| **Project Manager** | Coordinates team, manages backlog, organizes meetings |
| **Backend Developer / Logic Developer** | Implements application logic, integrates database and APIs |
| **Frontend Developer / UI Designer** | Designs and implements the user interface |
| **Database / Data Specialist** | Designs data model, manages database structure |
| **QA & Documentation Manager** | Performs testing, prepares documentation |

In the Low-Code track, roles may also include **Automation Designer** and **AI Integration Specialist**.

---

## Assessment Criteria

| Component | Points |
|---|---|
| Project repository / project environment | 10 |
| Project management plan (Jira, GitHub Projects, Trello) | 10 |
| Technical documentation | 10 |
| Team activity and commit history | 10 |
| Final presentation | 10 |
| **Maximum score** | **50** |

Additional aspects evaluated in the Low-Code track may include: system integrations, automation workflows, and use of AI tools.

## Project Timeline (12 Weeks)

### Week 1 — Project Kick-off
**Objectives:** Form teams, select a project idea 
**Deliverables:** Project description, repository created, initial MVP feature list

### Week 2 — Requirements Analysis
**Objectives:** Define system functionality 
**Deliverables:** Project backlog, defined MVP scope

### Week 3 — System Architecture Design
**Objectives:** Design the system architecture 
**Deliverables:** Architecture diagram, system design description

### Week 4 — Technical Setup
**Objectives:** Prepare the development environment 
**Deliverables:** Functional project skeleton

### Week 5 — Data Model Design
**Objectives:** Design the application data structure 
**Deliverables:** Data model diagram

### Week 6 — Core Logic Implementation
**Objectives:** Implement basic application functionality 
**Deliverables:** Core application functionality (API endpoints / workflows)

### Week 7 — User Interface Development
**Objectives:** Implement the user interface 
**Deliverables:** Basic user interface connected to backend

### Week 8 — System Integration
**Objectives:** Integrate all system components 
**Deliverables:** First working version of the application

### Week 9 — Authentication and AI Features
**Objectives:** Implement advanced features 
**Deliverables:** User authentication, roles, optional AI integration

### Week 10 — Testing
**Objectives:** Ensure application quality 
**Deliverables:** Stable application version

### Week 11 — Documentation
**Objectives:** Prepare project documentation 
**Deliverables:** README file, technical documentation

### Week 12 — Final Presentation
**Objectives:** Present the project results 
**Deliverables:** Final presentation, fully functional application

---

## Project Deliverables

Each team must submit:

- **Project Repository** — Application code or platform configuration with full commit history
- **Project Plan** — GitHub Projects, Jira, or Trello board showing task management
- **Documentation** — System architecture, technology overview, installation and usage instructions
- **Final Presentation** — 10–15 minute live demo of the application

---

## Recommended Development Methodology

Teams are encouraged to use **Agile / Scrum** practices with a typical bi-weekly sprint cycle:

1. Sprint Planning
2. Development
3. Code Review / Testing
4. Merge / Deployment
5. Sprint Demo

The project develops the following competencies:

- Software system design
